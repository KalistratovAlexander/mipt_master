## СЛАЙД 1: Эволюция рекомендательных систем — три поколения

### От коллаборативной фильтрации к генеративным моделям

```
┌──────────────────────┐   ┌──────────────────────┐   ┌──────────────────────┐
│  Поколение 1         │   │  Поколение 2         │   │  Поколение 3         │
│  Collaborative       │──>│  Deep Learning       │──>│  Generative          │
│  Filtering           │   │  (Embeddings)        │   │  Recommendation      │
│                      │   │                      │   │                      │
│ Matrix Factorization │   │ GRU4Rec (2016)       │   │ TIGER (NeurIPS 2023) │
│ Item-KNN, BPR        │   │ SASRec (2018)        │   │ PLUM (Google, 2025)  │
│                      │   │ BERT4Rec (2019)      │   │ OpenOneRec (Kuaishou)│
│                      │   │                      │   │ OneRec (Kuaishou)    │
│ Проблемы:            │   │ Проблемы:            │   │ TRM (ByteDance, 2026)│
│ • cold start         │   │ • случайные ID без   │   │                      │
│ • разреженность      │   │   семантики          │   │ Впервые:             │
│ • нет контента       │   │ • нет обобщения      │   │ • понимание контента │
│                      │   │ • embedding-таблицы  │   │ • multi-task         │
│                      │   │   = 99.6% параметров │   │ • cold-start решён   │
│                      │   │   (PLUM, YouTube)    │   │ • объяснимость       │
└──────────────────────┘   └──────────────────────┘   └──────────────────────┘
```

**Контекст:** PLUM (Google DeepMind) показал, что в продакшн-модели YouTube на embedding-таблицы приходится ~99.6% всех параметров, а нейросеть — лишь 0.4%.

> **Обзор Liu et al. (KDD 2024):** Авторегрессивная обработка последовательностей *естественно* соответствует задаче рекомендаций — пользовательские поведенческие паттерны можно моделировать как задачу генерации.

## СЛАЙД 2: Проблема представления товаров — центральный Research Gap

### Как «назвать» товар для модели? Четыре парадигмы

| Подход | Представители | Представление товара | Ключевая проблема |
|--------|--------------|---------------------|-------------------|
| **Случайный ID** | SASRec, BERT4Rec, GRU4Rec | item_42981 | Нет семантики, ID drift (Meta: 50% айтемов уходят за 6 дней) |
| **Текст** | P5, InstructRec, LLM-as-RS | Полное название/описание | Огромные контексты, дорогой инференс |
| **Collaborative ID** | CID (Hua et al.) | ID из поведенческих данных | Теряет контентные признаки |
| **Semantic ID** | TIGER, PLUM, OpenOneRec, OneRec | Иерархический дискретный код | **Наш подход** |

### Фундаментальное противоречие:

```
Компактность ◄──────────── TRADE-OFF ────────────► Семантика
(3-4 токена)                                        (1024d вектор)

  Случайный ID                      Плотный embedding
  ✓ Компактно                       ✓ Семантически богат
  ✗ Нет семантики                   ✗ Не дискретен
  ✗ ID drift (Meta)                 ✗ Потеря memorization (Singh et al.)

                     Semantic ID
                 ✓ Компактно (3-4 токена)
                 ✓ Семантически осмыслен
                 ✓ Совместим с LLM
                 ✓ Стабилен (Singh: 6 мес без деградации)
```

Singh et al. (Google, 2024) показали: прямая замена ID на плотные embeddings **ухудшает** качество из-за потери memorization. Semantic ID — компромисс, сохраняющий оба свойства.

Meta (Zheng et al., 2025) подтвердили: Semantic ID создаёт стабильное пространство с **семантически осмысленными** коллизиями вместо случайных, решая проблемы ID drift и impression skew.

## СЛАЙД 3: TIGER — основополагающая работа (NeurIPS 2023)

### Recommender Systems with Generative Retrieval

**Авторы:** S. Rajput, N. Mehta et al. (Google DeepMind). **Первая** генеративная модель на Semantic ID для рекомендаций.

**Ключевая идея:** Рекомендация = генерация последовательности токенов.

```
Текст товара ──> Sentence-T5 (768d) ──> RQ-VAE ──> (c₁, c₂, c₃)
                                          │            Semantic ID
                                     3 уровня × 256 кодов

Encoder-Decoder (T5X, ~13M параметров)
Вход: [user_token, SID₁, SID₂, ..., SIDₙ]
Выход: SID_{n+1} (авторегрессивно)
```

**Результаты на Amazon:**
| Метрика | SASRec | S³-Rec | **TIGER** | Улучшение |
|---------|--------|--------|-----------|-----------|
| Beauty R@5 | 0.0387 | 0.0387 | **0.0454** | +17.3% |
| Beauty N@5 | 0.0249 | 0.0249 | **0.0321** | +29.0% |
| Toys N@5 | 0.0306 | 0.0294 | **0.0371** | +21.2% |

**Новые возможности:** Cold-start рекомендации, управление разнообразием через температуру. Невалидные ID: 0.1-1.6%.

**Выявленные ограничения (в последующих работах):**
- Модель с нуля (~13M) — не использует pre-trained знания LLM
- Только sequential recommendation, нет multi-task
- OpenOneRec: -10.6% при joint multi-domain training
- PROMISE (Kuaishou): каскадные ошибки при генерации (Semantic Drift)
- SpecGR (AAAI 2026): near-zero recall на unseen items

## СЛАЙД 4: PLUM — промышленное внедрение на YouTube (Google DeepMind, 2025)

### Adapting Pre-trained LLMs for Industrial-scale Generative Recommendations

**Авторы:** R. He, N. Mehta et al. (23 автора). Развёрнуто в **продакшене YouTube**.

**Трёхстадийный pipeline:**
```
Стадия 1: SID-v2 (улучшенный RQ-VAE)
├── Мультимодальные эмбеддинги (текст + визуал + аудио)
├── Мультирезолюционные кодбуки: 2048/1024/512/...
├── Прогрессивная маскировка (каждый префикс осмыслен)
└── Контрастивная co-occurrence loss (+1.8% Recall@10 вклад)

Стадия 2: Continued Pre-Training (CPT)
├── 50/50: пользовательское поведение + метаданные
├── 1M шагов, ~260 млрд токенов
└── Без CPT: Recall@10 падает с 0.28 до 0.23 (-18%)

Стадия 3: SFT для генеративного поиска
├── ~100 последних просмотров, 1536 токенов
├── Взвешенная CE с reward-сигналом
└── Hallucination rate < 5%
```

**Результаты онлайн A/B на YouTube:**
| Метрика | Длинное видео | Shorts |
|---------|-------------|--------|
| Вовлечённые пользователи | +0.07% | **+0.28%** |
| Panel CTR | +0.76% | **+4.96%** |
| Просмотры | +0.80% | +0.39% |

Эффективный словарь PLUM в **2.6-13.2x** больше → лучшее покрытие каталога. Обучение на **<0.55x FLOPs** по сравнению с baseline.

## СЛАЙД 5: OpenOneRec — открытый фреймворк (Kuaishou, 2026)

### Open Foundation Model and Benchmark for Generative Recommendation

**Авторы:** G. Zhou et al. (47 авторов). **RecIF-Bench**: первый комплексный бенчмарк (8 задач, 3 домена, ~120M взаимодействий).

```
RQ-KMeans (3 уровня × 8192 кодов) → Itemic Tokens → Qwen3 (1.7B / 8B)
    ↓
Pre-training: Stage 1 (alignment, lr=1e-3) → Stage 2 (co-PT, 62/38% mix)
    ↓
Post-training: SFT → On-policy Distillation → Rec-RL (GRPO)
```

**Результаты:**
| Метрика | Лучший baseline | OneRec-8B-Pro | Улучшение |
|---------|----------------|---------------|-----------|
| Short Video Rec (Pass@32) | 0.1306 | **0.2122** | +62% |
| Amazon Transfer (avg R@10) | — | — | **+26.8%** |
| Few-shot (10% данных) vs TIGER | — | — | **+219.7%** |

**Scaling Laws:** N_opt ~ C^0.44, D_opt ~ C^0.56 → **data-intensive scaling** (данные важнее размера модели). Irreducible loss = 0.42 (vs 1.69 для текста).

**Сохранение общих способностей:** 98% MATH-500, 100% GSM8K при 62/38 data mix.

## СЛАЙД 8: Эволюция методов токенизации — от RQ-VAE к обучаемым токенам

### Хронология развития (2023-2026)

```
2023: TIGER ─── RQ-VAE (фиксированный, 2-stage)
       │
       ▼
2024: LETTER (CIKM) ─── + collaborative regularization + diversity loss
      CAGE (WWW) ────── дифференцируемое каскадное VQ (pluggable)
      CoFiRec ────────── coarse-to-fine иерархия
      CoST ───────────── контрастивное квантование
       │
       ▼
2025: ETEGRec (SIGIR) ── end-to-end обучаемая токенизация
      PIT (Kuaishou) ──── динамический персонализированный токенизатор
      Diff. SID ────────── дифференцируемая релаксация квантования
      UniGRec ──────────── мягкие (soft) идентификаторы
      SIIT (Snap) ──────── self-improving итеративная коррекция
      ActionPiece (ICML) ── BPE-контекстная токенизация действий
       │
       ▼
2026: QARM V2 ────────── Res-KmeansFSQ (коллизии 78%→32%)
      OneMall ─────────── Res-KMeans + FSQ (коллизии 36%→11%)
      UniSID (Tencent) ── end-to-end через MLLM
      TRM (ByteDance) ─── Gen-tokens + Mem-tokens (BPE, 20M)
```

### Ключевая полемика: фиксированные vs обучаемые токены

| Аспект | Фиксированные (TIGER, PLUM) | Обучаемые (ETEGRec, PIT) |
|--------|----------------------------|--------------------------|
| Токенизация | Отдельный этап (RQ-VAE) | Совместно с рекомендацией |
| Преимущества | Простота, стабильность | Лучшее качество (+5-15%) |
| Недостатки | Потеря информации при квантовании | Сложность обучения, нестабильность |
| Промышленное применение | YouTube, Meta, Kuaishou | Kuaishou (PIT: A/B +0.4%) |

**SIIT** (Snap Inc.) предлагает компромисс: начать с фиксированных SID, затем итеративно улучшать через self-alignment с LLM (+8% в среднем, plug-and-play).

## СЛАЙД 10: Специфические проблемы Semantic ID и их решения в литературе

### Выявленные проблемы и предложенные решения

| Проблема | Кто выявил | Решение | Кто решил |
|----------|-----------|---------|-----------|
| **Коллизии кодов** (несколько товаров → один SID) | TIGER (добавляет 4-й токен) | FSQ на последнем уровне | OneMall: 36%→11%, QARM V2: 78%→32% |
| **Semantic Drift** (ошибка на раннем уровне каскадно накапливается) | PROMISE (Kuaishou) | Process Reward Model + PRM-guided beam search | PROMISE: +42% Recall |
| **Unseen items** (модель не генерирует SID новых товаров) | SpecGR (AAAI 2026) | Speculative decoding: drafting + verification | SpecGR: +20.6% R@50 |
| **Потеря информации при квантовании** | Liu et al. (Snap) | Инъекция плотных эмбеддингов минуя RQ-VAE: +5% R@5 | — (открытая проблема) |
| **Несогласованность с LLM** | SIIT (Snap) | Self-improving итеративная коррекция | SIIT: +8% в среднем |
| **Catastrophic forgetting** | OpenOneRec | Data mixing (62% general + 38% rec) | OpenOneRec: 98% MATH-500 |
| **ID drift** (50% айтемов уходят за 6 дней) | Meta | SID = стабильное семантическое пространство | Meta: -43% A/A variance |
| **Impression skew** (0.1% head = 25% показов) | Meta, COINS | SID трансферит CF-сигналы к cold items | COINS: +9.6% cold-start orders |

## СЛАЙД 12: Two-stage Training — консенсус литературы

### Все ведущие работы сходятся на многостадийном обучении

```
              TIGER    PLUM       OpenOneRec   OneRec     Наша работа
              (2023)   (2025)     (2026)       (2025)
─────────────────────────────────────────────────────────────────────
Квантование   RQ-VAE   SID-v2     RQ-KMeans    Bal.RQ-KM  RQ-VAE
              3×256    мультирез. 3×8192       3×8192     4×256

Stage 1       —        CPT        Itemic-Text  —          Vocab
(выравнив.)   (с нуля) (50/50)    Alignment               Expansion
                       1M steps   (frozen)                 lr=1e-3
```

## СЛАЙД 13: Test-time scaling — новое направление (PROMISE, Kuaishou)

### Process Reward Models for Generative Recommendations

**Проблема Semantic Drift:**
```
Правильный путь:  A₁₂₃ → B₄₅ → C₆₇ → D₈₉  ✓ Целевой товар
                   │
Ошибка на уровне A: A₅₅₅ → B?? → C?? → D??  ✗ Нерелевантное подпространство
                   ↑
          Каскадная ошибка — нет механизма исправления
```

**Решение PROMISE:** Process Reward Model (PRM) оценивает промежуточные шаги.

**Результаты:**
- Amazon Beauty: **+42.4% Recall@5**, +44.4% NDCG@5
- Kuaishou (A/B): +0.131% total app usage time
- PRM добавляет лишь ~15% параметров и ~10% латентности
- **Test-time scaling law:** увеличение beam width монотонно улучшает метрики без увеличения модели

> Это открывает новое направление: вместо увеличения модели — увеличение вычислений при инференсе.

## СЛАЙД 14: Полемика — Semantic ID vs LLM-as-RS (Snap Inc., 2025)

### Understanding Generative Recommendation from a Model-scaling View

**Главный тезис:** SID-based подход имеет **фундаментальный потолок** масштабирования.

```
SID-based GR:                        LLM-as-RS:
RS: 336K→192M → плато при 13M       Qwen3: 0.6B→14B → нет плато
LLM-энкодер: 77M→11B → нет эффекта  LoRA: ~1% параметров
Токенизатор: 3×256 → оптимум        Constrained beam search (Trie)

L(N_SI, N_CF) = E + A/N_SI^a + B/N_CF^b
Узкое место: N_SI ограничен SID     Нет информационного бутылочного горлышка
```

| Подход | Beauty R@10 | Sports R@10 | Cold-start R@10 |
|--------|------------|-------------|-----------------|
| SID-based (лучший) | ~0.065 | ~0.040 | 0.018 |
| **LLM-as-RS (14B)** | **~0.078** | **~0.048** | **0.030** |

**Однако:** LLM-as-RS генерирует полные названия → **дорогой инференс**. GR4AD (Kuaishou) подтвердил: LazyAR декодер даёт ~2x throughput для SID, невозможный для текста.

**Наша позиция:** SID — **инженерно оптимальный** компромисс качество/стоимость. При бюджете H100 ~$10 — единственно реалистичный подход.

## СЛАЙД 18: Практические рекомендации — GRID Handbook (CIKM 2025, Best Paper)

**Авторы:** C.M. Ju, L. Collins et al. (Snap Inc.). Первый open-source фреймворк (GRID) + систематический бенчмарк.

| Constrained beam search | Обязателен | **Unconstrained не хуже, но быстрее** |

## СЛАЙД 19: ReSID — критика RQ-VAE и альтернативный подход (2026)

### Rethinking Generative Recommender Tokenizer

**Два фундаментальных недостатка RQ-VAE:**

1. **Несогласованность представлений:** Фундаментальные модели оптимизируют семантическое сходство, что конфликтует с коллаборативными сигналами. "Товары, часто совместно покупаемые, могут быть далеки в семантическом пространстве."

2. **Слабая предсказуемость последовательности:** RQ-VAE игнорирует требования авторегрессивного декодирования. "Один и тот же индекс может соответствовать разным семантическим направлениям под разными префиксами" → многомодальность.

**Решение ReSID:** FAMAE (Field-Aware Masked Auto-Encoding) + GAOQ (Globally Aligned Orthogonal Quantization)
- Результат: **>10% среднее улучшение** над SOTA
- **77-122x снижение вычислительных затрат** токенизации vs LETTER/RQ-VAE
- Впервые SID-метод превосходит ID-based модели с боковой информацией

> **Значимость:** Подтверждает, что RQ-VAE — не оптимальный, но рабочий метод. Наша работа фиксирует baseline на RQ-VAE, открывая путь для будущих улучшений (ReSID, GAOQ).
