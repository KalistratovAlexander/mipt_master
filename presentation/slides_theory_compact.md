## СЛАЙД 1: Проблема и ландшафт решений

**Проблема:** Как представить товар для LLM-рекомендаций?

| Подход | Пример | Плюс | Минус |
|--------|--------|------|-------|
| Случайный ID | item_42981 | Компактно | Нет семантики, ID drift [28] |
| Полный текст | "Purina Pro Plan..." | Богатая семантика | Дорогой инференс, на 20% лучше но в 10x медленнее [10] |
| **Semantic ID** | `<A12><B95><C195><D0>` | **Компактно + семантично** | Потеря информации при квантовании [10] |

**Semantic ID** — 3-4 дискретных токена из RQ-VAE, кодирующих иерархию: категория → подкатегория → группа → товар. Cos_sim между товарами с общим префиксом: 0.41 → 0.68 → 0.91 → 0.97 [18].

Подход валидирован в продакшене YouTube (+4.96% CTR) [5], Kuaishou (+14.71% GMV) [25], ByteDance (-33% storage) [27], Meta (top feature >1 года) [28] — суммарно 3+ млрд пользователей.

## СЛАЙД 2: Полемика — что работает, а что нет

| Тезис | За | Против |
|-------|----|--------|
| SID полностью заменяет классический pipeline | OneRec [2]: +1.68% Watch Time, OPEX -90% | Liu et al. [10]: потолок масштабирования, LLM-as-RS на 20% лучше |
| RQ-VAE — лучший метод квантования | TIGER [17]: +29% vs SASRec | GRID [8]: RQ-KMeans >= RQ-VAE при меньшей сложности; Spotify [16]: RQ-VAE нестабилен при multi-task |
| Фиксированные SID достаточны | PLUM [5], Meta [28]: работают на масштабе | ETEGRec [9]: end-to-end +5-15%; PIT [19]: динамические SID, A/B +0.4% |
| Генерация SID надёжна | TIGER [17]: hallucination 0.1-1.6% | PROMISE [4]: Semantic Drift — каскадные ошибки; решение: PRM (+42% Recall) |

**Консенсус:** двухстадийное обучение (alignment → finetune) критично. Без Stage 1: -18% Recall [5], -6% AUC [29].

## СЛАЙД 3: Путь к исследовательскому вопросу и гипотезам

**Пробелы в литературе:**
- TIGER [17] — модель 13M с нуля, не использует pre-trained LLM
- PLUM [5] — закрытая модель Google, 1024 TPU, невоспроизводимо
- OpenOneRec [29] — 5 стадий, 47 авторов, 8192 кодов — избыточно сложно
- Нет простого воспроизводимого pipeline для малых команд

**Исследовательский вопрос:** Как адаптировать открытые LLM (Qwen3) для товарных рекомендаций через Semantic ID на доступном оборудовании?

| Гипотеза | Обоснование |
|----------|-------------|
| **H1:** RQ-VAE создаёт семантически значимую иерархию | Singh [18]: cos_sim 0.41→0.97 на YouTube (~100M видео) |
| **H2:** Two-stage finetune → hallucination < 10% | PLUM [5]: <5%, TIGER [17]: 0.1-1.6% |
| **H3:** LLM-подход превосходит SASRec baseline | TIGER [17]: +17-29%, OpenOneRec [29]: +26.8% R@10 |

## СЛАЙД 4: Компоненты проблемы и позиционирование

| Теоретический | Практический |
|---------------|-------------|
| Может ли LLM освоить SID через vocab expansion? [29]: да | 3-stage pipeline: RQ-VAE → Vocab Exp → Full FT |
| Сохраняются ли общие способности? [29]: 98% MATH-500 | Amazon Pet Supplies: 63K товаров, 102K последовательностей |
| Осмыслена ли иерархия RQ-VAE? [18]: подтверждено | Qwen3-1.7B vs 8B: стоимость vs качество |
| Подтверждается ли потолок SID? [10] | 10+ задач, 20+ метрик; H100 на Vast.ai ~$10 |

**Позиционирование:** между TIGER (13M, академический) и PLUM (900M, промышленный). Воспроизводимый pipeline на открытых моделях с комплексной оценкой.

**База:** 31 источник, 9 на топ-конференциях (NeurIPS, ICML, AAAI, KDD, SIGIR, RecSys, WWW, CIKM), 74% опубликованы в 2025-2026.
