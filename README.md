## Аннотация

В данной работе исследуется применение иерархических семантических идентификаторов (Semantic IDs) для генеративных товарных рекомендаций. Каждому товару каталога присваивается уникальный многоуровневый код, полученный с помощью Residual Quantized VAE (RQ-VAE), что позволяет большой языковой модели генерировать идентификаторы товаров непосредственно как последовательности токенов. Подход оценивается на датасете Amazon Pet Supplies (63 319 товаров, 101 926 пользовательских последовательностей) в двух предзарегистрированных экспериментах: **H1** — задачно-условный эффект масштаба базовой модели (Qwen3 0.6B / 1.7B / 4B / 8B), **H2** — влияние способа инициализации эмбеддингов добавленных токенов (четыре способа при равной норме Фробениуса, Qwen3-0.6B). Значимость оценивается парным бутстрепом (10 000 выборок) с поправкой Бонферрони.

## Метод

Система состоит из трёх этапов:

1. **Назначение Semantic ID.** Эмбеддинги товаров (Qwen3-Embedding-0.6B, 1024d) квантуются трёхуровневым RQ-VAE с кодовыми книгами размером 256 и rotation trick для оценки градиентов. Детерминированный четвёртый уровень (D) устраняет коллизии внутри групп (A,B,C), обеспечивая уникальные 4-уровневые идентификаторы для всех 63 319 товаров.

2. **Расширение словаря.** В токенизатор LLM добавляются 1 027 специальных токенов (`<|sid_start|>`, `<|sid_end|>`, `<|A0|>`–`<|D255|>`). На данном этапе обучаются только матрицы эмбеддингов (Stage 1).

3. **Полное дообучение.** LLM обучается на 4,97 млн диалогов в формате инструкций, охватывающих задачи сопоставления текста и SID, последовательных рекомендаций и предсказания совместных покупок (Stage 2). Обучение использует Unsloth для загрузки модели, маскирование инструкций, упаковку последовательностей (~3x ускорение) и cosine-with-minimum-LR планировщик.

Все конфигурации обучаются с идентичными гиперпараметрами (LR=2×10⁻⁵, warmup 3%, weight decay 0.01, эффективный батч 128, 3 эпохи) для контролируемого сравнения.

## Результаты

Оценка на едином наборе из четырёх категорий задач (генерация идентификатора по тексту, предсказание следующего и сопутствующих товаров, генерация текста по идентификатору) по метрикам Recall@10 и косинусного сходства. NVIDIA H100 80GB.

### H1. Задачно-условный эффект масштаба (Qwen3 0.6B / 1.7B / 4B / 8B)

Recall@10 на задаче генерации идентификатора по названию:

| Уровень иерархии | 0.6B | 1.7B | 4B | 8B |
|---|---|---|---|---|
| Уровень A | 59.9% | 62.0% | 71.9% | 74.8% |
| Полный ABCD | 1.5% | 1.0% | 2.5% | 1.7% |

Значимый прирост сосредоточен в переходе **1.7B → 4B** — единственном контрасте, значимом на всех шести семантических подзадачах. Переход 4B → 8B не даёт значимого прироста ни на одной задаче, но повышает энергозатраты на 67% и снижает пропускную способность на 21%. На коллаборативных задачах 14 из 15 попарных контрастов незначимы. Оптимум по совокупности качества и стоимости — **Qwen3-4B**.

### H2. Влияние способа инициализации эмбеддингов (Qwen3-0.6B, 4 способа × 3 seed)

Способы инициализации добавленных токенов при выравненной норме Фробениуса: **A** — ковариационная гауссова, **B** — изотропный контроль, **C** — усреднение токенов названия товара, **D** — проекция кодовой книги RQ-VAE.

| Метрика | A | B | C | D |
|---|---|---|---|---|
| Recall@10 ABCD (по названию) | 1.0% | 1.1% | 1.6% | 1.0% |
| cosine_sim (генерация названия) | 0.464 | 0.421 | 0.494 | 0.445 |
| Перплексия WikiText-2 (кратность ухудшения) | 21× | 24× | 40× | 123× |

Способ **C** значимо превосходит A на всех трёх задачах генерации идентификатора (p от 0.0002 до 0.0090). Способ A статистически неотличим от контрольного B (p = 0.710): направленный сигнал сам по себе качества не добавляет. Способ D деградирует катастрофически (перплексия ×123) при геометрии, близкой к C, — одной близости к кодовой книге недостаточно, нужен лингвистически осмысленный сигнал. Рекомендуемая инициализация — **C**.

## Воспроизведение

### Окружение

```bash
pip install -r requirements.txt
```

Для обучения на GPU необходимо установить PyTorch с поддержкой CUDA:
```bash
pip install torch --index-url https://download.pytorch.org/whl/cu121
```

### Подготовка данных

Ноутбуки выполняются последовательно от `data/download_data.ipynb` до `pipeline/prepare_data_for_llm.ipynb`. Шаги 2–3 требуют ключ Gemini API; шаг 7 (генерация эмбеддингов) выполняется в среде Kaggle с GPU.

### Обучение

Скрипты обучения предназначены для развёртывания на [vast.ai](https://vast.ai) с GPU NVIDIA H100 80GB.

```bash
# Сборка пакета для развёртывания (масштаб: 0.6b | 1.8b | 4b | 8b)
bash pipeline/fine_tune_h1/pack.sh 8b

# Загрузка на удалённый GPU-инстанс
scp vast_8b_package.tar.gz root@<HOST>:/workspace/

# На удалённом инстансе
cd /workspace && tar xf vast_8b_package.tar.gz
export HF_TOKEN=hf_...
bash run_smoke.sh     # smoke-тест (~10 мин)
bash stage1/run.sh    # Stage 1: ~1 час
bash stage2/run.sh    # Stage 2: ~10 часов
```

Остальные масштабы собираются той же командой с аргументом `0.6b` / `1.8b` / `4b`.

### Оценка

```bash
python pipeline/evaluation/evaluate_unified.py \
    --model-path models/qwen3_8b_sid \
    --data-dir data
```
